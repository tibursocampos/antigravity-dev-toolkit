# Operational and Developer Skills

These skills are used outside or around the full SDD lifecycle.

## Main operational chain

- `code_review`: review quality and risk before merge.
- `repair_dotnet_build`: recover broken local build/test state.
- `test_coverage`: measure coverage against thresholds.
- `commit`: create approved conventional commits.
- `push`: publish branch safely.

All mutating git actions still require explicit confirmation.

## Focused implementation skills

- `developer`: direct small/medium implementation path.
- `refactor`: safe structural refactoring path.
- `api_integrate`: API client integration generation.
- `performance_profile`: profiling and performance optimization.
- `containerize`: Docker and compose scaffolding.
- `i18n_manager`: localization extraction and replacement.

## Backlog preparation skills

- `refine_story`: convert informal requests to structured backlog markdown with BDD criteria.
- `split_story_checklist`: convert refined backlog into grouped implementation task checklists.

These two skills are upstream preparation and naturally hand off to `sdd_spec` / `sdd_plan` / `sdd_develop`.

## Language and naming

- Use underscore skill names in docs and examples.
- Keep chat in `pt-BR`.
- Keep generated code/tests and skill files in English.
