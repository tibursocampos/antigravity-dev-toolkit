# Shared Guidelines and Conventions

The Antigravity Dev Toolkit utilizes a set of shared guidelines to ensure consistency across all skill executions. These shared documents are stored in the `plugin/skills/_shared/` directory and are lazily loaded by individual skills only when their specific context is required.

## Shared Directories

### 1. `_shared/dotnet_guidelines`
Contains architectural rules and coding patterns specifically for .NET projects. This ensures that any C# code generated by the agent aligns with enterprise standards.
- **Key Files**: 
  - `clean-architecture.md`: Enforces separation of concerns (Domain, Application, Infrastructure, API).
  - `csharp-patterns.md`: Defines preferred C# language features, LINQ usage, and test structures.
  - `checklist.md`: A pre-commit and pre-review checklist to catch common omissions.

### 2. `_shared/code_guidelines`
Contains general software engineering principles that apply regardless of the specific programming language.
- **Key Files**: Includes a `principles/` folder for fundamental rules (like SOLID, DRY, and KISS) referenced during the `code_review` and `sdd_develop` phases.

### 3. `_shared/sdd_artifacts`
Governs the lifecycle, storage, and validation of the PRD and PLAN artifacts within the repository.
- **Key Files**:
  - `PIPELINE.md`: Defines the state machine rules (e.g., you cannot plan without a PRD, you cannot implement without a PLAN).
  - `STORAGE.md`: Defines where artifacts are stored (supporting local `/PRD/`, `/PLAN/` folders or external/global path resolution via manifest.json) and how they are numbered.

## Language Rules

The toolkit strictly partitions the languages used for communication and artifacts versus production code to maintain a clean codebase while ensuring comfortable interactions for the user.

| Context | Required Language |
|---------|-------------------|
| **Chat / Agent Replies** | Brazilian Portuguese (pt-BR) |
| **SDD Artifacts (PRD, PLAN)** | Brazilian Portuguese (pt-BR) |
| **Product Documentation (`docs/`)** | User's choice (asked before creation) |
| **Code (C#, JS, etc.)** | English |
| **Tests & Comments** | English |
| **Identifiers (Vars, Classes)** | English |
| **File paths (except PRD/PLAN text)**| English |

## Naming Conventions

Consistency in naming allows the toolkit to reliably auto-discover files using Glob patterns.

- **PRD Files**: `NNN_short_feature_slug.md` (e.g., `001_authentication.md`) under the active storage folder (local or global)
- **PLAN Files**: `PLAN_NNN_short_feature_slug.md` (e.g., `PLAN_001_authentication.md`) under the active storage folder (local or global)
- **Skill Directories**: `snake_case` (e.g., `developer`, `code_review`)
- **Skill Manifest Names**: `snake_case` inside the `SKILL.md` frontmatter (e.g., `name: developer`)
- **Git Branches**: `feature/<slug>` or `feat/<id>` (nested directories like `feature/a/b` are explicitly rejected by the `commit` skill).
