# DESIGN-BRIEF.md template (framework-agnostic)

Generated by `use skill impeccable shape` or the shape phase of `craft`. Consumed by `*-developer` skills and `developer` router. Do not reinterpret visual decisions during implementation.

**Output path:** `docs/DESIGN-BRIEF.md` (or `docs/design/DESIGN-BRIEF.md` when the repo already uses `docs/design/`).

---

## 1. Register and product context

- **Register:** `brand` | `product`
- **PRODUCT.md:** link or path
- **DESIGN.md:** link or path (if present)

## 2. Feature summary

What this is, who it is for, what it must accomplish (2-3 sentences).

## 3. Component tree / information architecture

High-level structure: pages, regions, components, data flow.

## 4. Design tokens

OKLCH palette roles, typography scale, spacing rhythm. Link to `DESIGN.md` when tokens live there.

## 5. State map

Per interactive control, document all eight states where applicable: default, hover, focus, active, disabled, loading, error, empty.

## 6. Accessibility checklist

Focus order, labels, contrast targets (4.5:1 body, 3:1 large text), keyboard paths, `prefers-reduced-motion`.

## 7. Anti-patterns explicitly avoided

List upstream bans the design refuses (e.g. glassmorphism default, gradient text, identical card grids).

## 8. Target stack

`react` | `angular` | `vue` | `blazor` | `maui` | `electron` | `html-css`

## 9. Implementation notes (optional)

Stack-specific mapping only when needed (e.g. Blip `blip-ds`, Angular standalone vs NgModule).

## 10. Implementation scope (one session)

Single deliverable for the next `*-developer` session: files, components, acceptance criteria.
